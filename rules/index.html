The rules

Sitespeed uses rules to decide if a page is optimized for performance. The rules are based on perfromance best practices, check out the exact rules used here.

Modified Yslow rules

Sitespeed uses many rules from Yslow (read about them here). The only yslow rule that isn't used by sitespeed, is the CDN rule. Also the tolerance for number of request is less with sitespeed (max 2 js, 2 css and 3 css images).

Do not load specific css for print

Loading a specific stylesheet for print, can block rendering in your browser (depending on version) and will for almost all browsers, block the onload event to fire (even though the print stylesheet isn't even used). You can read more about it here.

Load CSS in head from document domain

CSS files in head, should be loaded from the same domain as the main document, in order to avoid DNS lookups. This is extra important for mobile.

Never load JS synchronously in head

Javascript files should never be loaded synchronously in head, because it will block the rendering of the page. Alternative version to yjsbottom rule, that don't seems to work with phantomjs.

Avoid use of web fonts

Avoid use of webfonts because they will decrease the performance of the page. Read more.

There are also two rules borrowed from Stoyan Stefanov.

Load third party JS asynchronously

Always load third party javascript asynchronously. Third parties that will be checked are twitter, facebook, google (api, analythics, ajax), linkedin, disqus, pinterest & jquery. Read more about asynchronously third party scripts here.

Load the third party JS only once

Make sure the third party javascripts are loaded only once. More info here.
